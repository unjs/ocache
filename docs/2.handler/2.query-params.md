# Query Parameters

By default the full query string varies the cache key, so `?color=red` and `?color=red&utm=x` are cached separately and unknown params can bust the cache. Set `allowQuery` to an allowlist of param names so only those affect the key — all other params are ignored.

Ignored params are also stripped from the URL the handler receives (like non-`varies` headers), so a handler can never accidentally produce output that depends on a param outside the key. Param order is normalized, and repeated (array) params like `?color=red&color=blue` are matched regardless of order.

```ts
const handler = defineCachedHandler(myHandler, {
  maxAge: 300,
  allowQuery: ["color"], // ?color=red&lang=en and ?color=red&lang=de share one entry
});
```

## Edge Cases

- **Vary by nothing.** Passing an empty array (`allowQuery: []`) means every query string shares one entry.
- **Custom `getKey`.** If you set a custom `getKey`, it controls the key entirely and `allowQuery` no longer affects the key — but non-allowlisted params are still stripped from the URL the handler receives.
